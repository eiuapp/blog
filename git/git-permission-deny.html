gitlab-shell: No such file or directory

现象

╭─a at a in ~/sda3/Users/a/Desktop/tianluo/fangyuan/src/fangyuan-server on dev✔ using
╰─± git pull -v origin dev
bash: /opt/gitlab/embedded/service/gitlab-shell/bin/gitlab-shell: No such file or directory
fatal: Could not read from remote repository.

Please make sure you have the correct access rights
and the repository exists.

使用当前机器的另一用户尝试

把 ~/.ssh/id_rsa* 放在客户端机器的其它的用户(另创test用户)下，则 git clone 成功, 说明:

  • 与 key（~/.ssh/id_rsa*）　无关。
  • 与　服务端　无关，只与客户端本机相关。
  • 只与客户端本机的当前用户相关。

(可跳过)

  • https://wiki.shileizcc.com/confluence/display/gitlab/Gitlab+Update+Version 搜索 gitlab-shell
