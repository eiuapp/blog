代码自动生成~swagger-ui~

bad

  • swagger2.0 将来会升级至 openapi3.0, 现在与代码集成了, 将来,其它团队升级至 openapi3.0, 你怎么办?
  • 与代码相关. 服务不启动, 就看不到对应的~swagger-ui~
  • 代码启动后, 如果服务对外开放, 就要对这个 URL 专门进行处理(比如:禁止访 问~swagger-ui~对应的地址)
  • 与代码相关. 代码背后是一种语言, 那么, 就失去了语言迁移.
  • 代码注释更多, 代码量显得多.

good

  • 代码注释更多, 理解代码更容易.

m1

  • step
    • swagger: 写文档
    • dredd: 进行测试
    • assertible: post deploy hook, 监控
    • redoc: generated API Reference Documentation
    • OpenAPIValidators: 对openapi responses进行 validate

    https://github.com/Redocly/redoc https://github.com/RuntimeTools/OpenAPIValidators

  • bad

    dredd 写的测试,必须在 api.yaml 中加入 example annotation, 这样, 我们要测试的案 例肯定不全.
