注意 openapi 写法

swagger 2.0

来自 https://github.com/swagger-api/swagger-node/blob/master/docs/mock-mode.md#quick-example-mock-mode-in-action 的示例 

swagger: '2.0'
info:
  version: "0.0.1"
  title: Mock mode test
host: localhost
basePath: /
schemes:
  - http
consumes:
  - application/json
produces:
  - application/json
paths:
  /weather:
    x-swagger-router-controller: weather
    get:
      responses:
        "200":
          description: Success
          schema:
            $ref: "#/definitions/WeatherResponse"
definitions:
  WeatherResponse:
    required:
      - message
    properties:
      message:
        type: string

命名成 weather.yaml, 运行 

prism mock -d ./weather.yaml
curl -i http://localhost:4010/weather

会正常返回 mock 数据了。

openapi 3.0

改写上面的 swagger 2.0 为 openapi 3.0 

openapi: 3.0.0
info:
  title: weather
  description: weather project
  version: 0.0.1

servers:
  - url: http://192.168.168.165:7014
    description: Optional server description, e.g. Internal staging server for dev

paths:
  /string:
    x-swagger-router-controller: weather
    get:
      responses:
        '200':
          description: Success
          content:
            application/json:
              schema:
                required:
                  - message
                properties:
                  message:
                    type: string

注意： responses 必须写上 

content:
  application/json:

否则，~curl -i http://localhost:4010/weather~ 会无法得到 mock 数据。

主要是２个原因：

  1. security 设置不正确
  2. prism -d　失效，手工重启

如果不要 security, 则在yaml中设置为 [] 

post:
  security: []
  responses:
    '200':
      content:
        application/json:
