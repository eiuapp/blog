写成的 *.yaml 文件，请确保语法正确

  • https://editor.swagger.io/ 中输入后无提示出错

比如：

schema: '#/components/schemas/ExampleSchemaObject' 这个写法，不支持，会报错 ~throw new Error(`Invalid OpenAPI spec: ${error.message}`);~。 

throw new Error(`Invalid OpenAPI spec: ${error.message}`);
          ^
Error: Invalid OpenAPI spec: [ { keyword: 'type',
    dataPath:
     '.paths[\'/example\'].get.responses[\'200\'].content[\'application/json\'].schema',
    schemaPath: '#/type',
    params: { type: 'object' },
    message: 'should be object' },

要换成 

              schema:
                $ref: '#/components/schemas/ExampleSchemaObject'
components:
  schemas:
    ExampleSchemaObject:
      type: object

语法正确情况下， paths 中使用 $ref 则也会提示 AssertionError: expected res to satisfy API spec
